## 2. 移管作業 A: Supabase（最重要）

現在のSupabaseプロジェクトは**開発者の個人契約**。単会（事務局）契約へ移す方法は2通り。

### 方法1: プロジェクトの所有権移管（推奨・データそのまま）
1. 事務局のメールアドレスで Supabase アカウントを作成
2. 事務局で Supabase の Organization（組織）を作成
3. 現所有者が Supabase ダッシュボード →対象プロジェクト → Settings → General → "Transfer project" で事務局Organizationへ移管
4. 移管後、課金プラン（無料枠で足りる想定）を事務局側で設定
5. **アプリ側の変更は原則不要**（プロジェクトURL・キーが変わらなければそのまま）

### 方法2: 新規プロジェクトを作り直してデータ移行
1. 事務局アカウントで新規 Supabase プロジェクト作成
2. テーブル作成（下記スキーマ参照）
3. 旧プロジェクトから Table Editor → Export、新プロジェクトへ Import
4. Storage バケット `speaker-files` を作成（Public 設定）し、旧バケットのファイルをダウンロード→アップロード
5. **アプリの接続先を変更**（`src/lib/supabase.js` の `SUPABASE_URL` と `SUPABASE_KEY` を新プロジェクトの値に書き換え → 再ビルド・再デプロイ）

### 必要なDBスキーマ（方法2の場合）

```sql
-- speakers テーブル
create table speakers (
  id text primary key,
  district_id uuid,
  chapter_id text,
  seminar_type text,
  speaker_name text,
  speaker_kana text,
  speaker_unit text,
  company text,
  role text,
  company_role text,
  seminar_date date,
  topic text,
  status text,
  phone text,
  email text,
  request_date date,
  notes text,
  venue text,
  line_notified boolean,
  material_url text,
  material_name text,
  lodging text,
  print_required text,
  speaker_checks jsonb default '{}',
  calendar_added boolean,
  drinks_alcohol text,
  shiori_article text,
  post_notes text
);

-- tasks テーブル
create table tasks (
  id text primary key,
  chapter_id text,
  district_id uuid,
  title text,
  due_date date,
  done boolean,
  priority text,
  completed_at timestamptz
);
```

- `district_id` の固定値: `11111111-1111-1111-1111-111111111111`
- Storage バケット名: `speaker-files`（Public、匿名アップロード可のポリシー）
- RLS（行レベルセキュリティ）: 現状は publishable key で読み書きできる設定。同等のポリシーを設定すること

### Storage内のファイル命名規則（現行）

```
{単会ID}/{開催日YYYYMMDD}/{講師名ローマ字}/{種別}.ext
例: niizashiki/20260729/kogushikazumi/photo.jpg
    niizashiki/20260729/kogushikazumi/doc1.pdf
種別: photo（顔写真） / doc1（講話資料①） / doc2（講話資料②）
```

## 3. 移管作業 B: GitHub / 公開URL

- リポジトリ: `https://github.com/hosina0447-ctrl/rinri-nanbu`
- 公開: `gh-pages` ブランチ + GitHub Pages
- 公開URL: `https://hosina0447-ctrl.github.io/rinri-nanbu/`

### 移管手順
1. 事務局のGitHubアカウントを用意
2. リポジトリの Transfer ownership（Settings → Danger Zone → Transfer）または事務局アカウントへ Fork
3. Settings → Pages で公開ブランチ `gh-pages` / フォルダ root を確認
4. **公開URLが変わる場合**（例: `https://{新アカウント}.github.io/rinri-nanbu/`）:
   - `vite.config.js` の `base: '/rinri-nanbu/'` はリポジトリ名が同じなら変更不要
   - 講師フォームのURL（管理画面で生成される `form.html` へのリンク）の `BASE` を新URLへ更新
     - 対象: `src/components/FormURLModal.jsx` 内 `const BASE = '...form.html'`
   - 再ビルド・再デプロイ
5. デプロイ方法: `npm run build` 後、`dist/` の中身を `gh-pages` ブランチへ配置（`.nojekyll` 空ファイルを必ずルートに含める。無いと `assets/` が公開されず画面が真っ白になる）

## 4. Googleドライブ連携の設定（未実装・将来対応）

> 現時点では未実装。事務局のGoogleアカウントで以下を設定すれば、講師の写真・資料を
> 事務局Googleドライブの指定フォルダへ自動保存できるようになります。

### 仕様（実装予定の動作）
- 管理画面で講師ごとに「📁 Driveに保存」ボタンを押す
- Googleアカウント選択ポップアップ → 事務局アカウントを選択（初回のみ利用許可）
- 自動でフォルダ **`2026年7月_新座志木_講師`** 形式（`{年}年{月}月_{単会}_講師`）を作成
- そのフォルダへ 顔写真・講話資料①②を保存（ファイル名は `{開催日}_{講師名}_{種別}.ext`）

### 事前設定（Google Cloud Console・無料・約10分）

> **費用は一切かかりません**。Drive API は無料。課金アカウント（クレジットカード）登録も不要。

1. [console.cloud.google.com](https://console.cloud.google.com/) に**事務局Googleアカウント**でログイン
2. 上部プロジェクト選択 → 「新しいプロジェクト」→ 名前「rinri-nanbu」→ 作成
3. 「APIとサービス」→「ライブラリ」→「Google Drive API」を検索 →「有効にする」
4. 「APIとサービス」→「OAuth同意画面」
   - User Type:「外部」→ 作成
   - アプリ名「倫理法人会 南部地区事務局」
   - ユーザーサポートメール／デベロッパー連絡先 = 事務局メール
   - スコープ・テストユーザー画面はそのまま「保存して次へ」
   - 公開ステータスを「本番環境に公開」にしておく（テストのままだと7日でログイン切れ）
5. 「APIとサービス」→「認証情報」→「+認証情報を作成」→「OAuth クライアント ID」
   - 種類:「ウェブアプリケーション」
   - **承認済みの JavaScript 生成元** に追加:
     `https://hosina0447-ctrl.github.io`
     （※GitHubアカウント移管で公開URLが変わったら、新しいオリジンに変更）
   - 作成
6. 表示される **クライアント ID**（`xxxx.apps.googleusercontent.com`）を控える
   - これは公開して問題ない値（パスワードではない）

### 実装担当者への引き継ぎ事項
- 上記クライアントIDをアプリに組み込む（`src/constants.js` 等に定数追加）
- 使用ライブラリ: Google Identity Services（GIS）トークンクライアント + Drive API `files.create`（multipart）
- スコープ: `https://www.googleapis.com/auth/drive.file`（アプリ作成ファイルのみ・最小権限）
- フォルダ検索→無ければ作成→ファイルupload の順。フォルダ名 `{年}年{月}月_{単会名短縮}_講師`
- 対象ファイル: `material_url`（顔写真）+ `notes` 内 `【資料01】`/`【資料02】` のURL
- 確認書（講師依頼確認書）は印刷→PDF保存方式のため、Drive自動保存の対象外
